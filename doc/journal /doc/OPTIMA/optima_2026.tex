	\section{Introduction} 
	The \textit{Longest Common Subsequence Problem} (LCSP)~\cite{DJUKANOVIC2020106499} is a well-known combinatorial optimization problem with numerous applications in   computational biology, particularly in the analysis of molecular  sequences. Given an arbitrary   set $S=\{s_1,\ldots,s_m\}$ of input sequences over an alphabet $\Sigma$, the goal is to determine the longest possible subsequence common to all $s_i \in S$. In recent decades, many practically motivated variants of the LCSP have been proposed by considering additional structural or biological constraints such as  the constrained, arc-preserving, repetition-free version, etc. This work is focused on solving the \textit{Variable Gapped} LCSP (VGLCSP), introduced in~\cite{penga2011longest} and theoretically examined in~\cite{adamson2023longest}. This problem extends the standard LCSP by imposing flexible distance limits (``gaps'') between consecutive symbols in the resulting subsequence, allowing these gaps to vary along the input sequences. Formally, for each input sequence $s_i \in S$, a gap function $G_{s_i}\colon \{1,\ldots,|s_i|\}\mapsto \mathbb{N}$ defines the maximum distance allowed between consecutive characters of a subsequence. That is, if the characters of a subsequence $s$ appear at the positions $i^1_i < \ldots < i^{{|s|}}_i$ in $s_i$, we say that the gap constraint is satisfied if $i^x_i - i^{x-1}_i \le G_{s_i}[i^x_i] + 1$ for all $x=2,\ldots,|s|$. A common subsequence $s$ is considered feasible if the corresponding gap constraints hold across all sequences in $S$. The VGLCSP represents a flexible and realistic model of sequence alignment, which is particularly useful for comparing DNA or protein sequences with variable structural distances. 	Although exact dynamic programming approaches exist for the two-sequence case ($m=2$), their computational complexity becomes prohibitive for larger $m$. To the best of our knowledge, no effective exact or approximate approaches currently exist for the generalized VGLCSP, a clearly NP-hard problem.
	
	Building on the well-established state-space representation of the LCSP~\cite{DJUKANOVIC2020106499}, we propose a general \textit{state graph} formulation for the VGLCSP. Each node represents a partial feasible subsequence, characterized by the vector of current positions in each input sequence and the subsequence length. Specifically, a state $v=(\mathbf{p}^{L,v}, l^v)$ is induced by a partial solution $s^v$ if: (i) $p^{L,v}_i-1$ is the smallest index such that $s^v$ is a subsequence of the prefix string $s_i[1]\cdots s_i[p^{L,v}_i-1]$; (ii) $l^v=|s^v|$; and (iii) all the gaps $G_{s_i}$ are preserved with $s^v$. A directed arc $\alpha=(v_1 v_2)$, labeled $lett(\alpha)={a} \in \Sigma$, exists between states $v_1$ and $v_2$ if $l^{v_2} = l^{v_1} + 1$ and $s^{v_2} = s^{v_1} \cdot {a}$. To expand a state $v$, all letters that occur in each suffix of $s_i$ defined by respective positions $p^{L,v}_i$  are considered. For each such letter $a$, the minimal positions $p^{L,a}_i \geq p^{L,v}_i$ of its appearances are identified fulfilling $p^{L,a}_i - p^{L,v}_i \le G_{s_i}(p^{L,a}_i)$. The resulting child node, if not suboptimal, is given by $w=(p^{L,a}+1, l^v+1)$. The root node  $r:=((1,\ldots,1),0)$ represents the empty subsequence, while goal states correspond to non-extendable feasible subsequences. Unlike the standard LCSP, the VGLCSP admits multiple (possibly exponentially many) root nodes. For example, in sequences $s_1=\texttt{ATGGAAAA}$ and $s_2=\texttt{ATCCAAAA}$ with $G_{s_1}=G_{s_2}=1$, the state $((1,1),0)$ cannot reach any state with the position vector $(5,5)$ via transitions, missing optimal subsequence $\texttt{AAA}$. This  motivates the methodology of \textit{Iterative Multi-Source Beam Search} (IMSBS). BS is a popular meta-heuristic approach that performs in a breadth-first-search manner, controlled by the size of beam $\beta$ and a heuristic $h$. In the IMSBS, BS is iteratively executed with a set of promising root nodes as an initial beam, selected from a dynamically maintained global pool of root nodes, controlling diversification of the approach. 
	Each iteration performs a complete BS, guided by a heuristic  $h$ based on letter frequencies, minimal residual substring lengths, or probability-weighted estimates.
	
 
 \subsection{Contributions}
 
 This work proposes a learning framework for constructing a data-driven heuristic to guide Beam Search in combinatorial optimization problems, and specifically one sequence problem that arises in bioinformatics, the \textit{Variable Gapped Longest Common Subsequence Problem} (VGLCSP). A feed-forward neural network (FFNN) is used to parameterize the heuristic function. Instead of relying on gradient-based training, the network weights are optimized using a Genetic Algorithm (GA), in particular its popular versions called the \textit{Random key generic algorithm} (RKGA) and a biased RKGA (BRKGA) approach.  
 
 Each individual in the GA population represents a complete set of neural network weights. The quality of an individual is evaluated by embedding the neural network inside the \textit{Iterative Multi-source Beam Search} (IMSBS) framework (proposed in the paper~\cite{}) to guide the search in the process of forward passes of the BS,  
 and measuring the average solution quality obtained on a set of training instances. Validation instances are used exclusively to monitor generalization.
 
 Formally, the learning problem is defined as:
 \[
 \max_{\theta} \; \frac{1}{|T|} \sum_{I \in T} f(\text{IMS\_BeamSearch}(I, h_\theta)),
 \]
 where $T$ denotes the training set, $h_\theta$ is the neural heuristic parameterized by weights $\theta$, and $f(\cdot)$ measures solution quality.

 \section{Utilizing Feed-Forward Neural Networks for Guiding the Search}
 
 The heuristic is represented by a \textit{multi-layer perceptron} with an arbitrary number of hidden layers and user-defined units per layer. Supported activation functions include \texttt{hyperbolic tangent}, \texttt{ReLU}, and \texttt{sigmoid}.
 
 For each layer $l$, forward propagation is defined as:
 \[
 \mathbf{z}^{(l+1)} = W^{(l)} \mathbf{a}^{(l)} + b^{(l)}, \quad
 \mathbf{a}^{(l+1)} = \phi(\mathbf{z}^{(l+1)}),
 \]
 where $W^{(l)}$ and $b^{(l)}$ are the weight matrix and bias vector, respectively, and $\phi$ denotes the activation function.
 
 All weights and biases are flattened into a single vector, which forms the chromosome of a GA individual.
 
 \subsection{Fitness Evaluation}
 
 Given a candidate weight vector:
 \begin{enumerate}
 	\item The weights are loaded into the FFNN.
 	\item IMSBS is executed on each training instance using the neural heuristic (guiding the search in the forward iterations as more sensible than the backward iterations).
 	\item The size of the best sequence obtained for each instance is recorded.
 \end{enumerate}
 
 The fitness value is computed as the average solution quality over all training instances. Validation performance is computed analogously on a separate dataset and is used only for reporting.

 \subsection{Feature Extraction for the Variable-Gap LCS}
 
 To guide the search process, each partial solution (node) is represented by a fixed-length numerical feature vector capturing both the current state of the construction and the structure of the remaining search space. These features are designed to be instance-independent and comparable across problem sizes.
 
 Let $S=\{s_1,\ldots, s_k\}$ denote the set of input sequences, and let $p_i$ be the current matched position in sequence $s_i$ at a given node. The vector $\mathbf{p}^L=(p_1,\ldots,p_k)$ is first normalized by the corresponding sequence lengths,
 \[
 p_i \leftarrow \frac{p_i}{|s_i|},
 \]
 so that progress in each sequence lies in $[0,1]$ and becomes independent of absolute input sizes.
 
 From this normalized position vector, four statistical descriptors are extracted: maximum, minimum, mean, and standard deviation (4 features). These quantities characterize, respectively, the most advanced and least advanced sequences, the average depth of the search, and the imbalance of progress among sequences. In addition, \textit{the current length of the partially constructed subsequence} is included as a feature (as the fifth feature). Together, these values provide a compact description of the current search state.
 
 To capture the feasibility of future extensions, gap-related features are computed from the positions of remaining sequences (subproblem) of input sequences. For each sequence $s_i$, all gap bounds associated with positions strictly after the current node position are collected.  \section{Genetic Algorithm}
 
 The optimization process follows a population-based evolutionary scheme.
 
 \subsection{Initialization}
 
 Each individual is initialized by sampling weights uniformly from a predefined interval [$-w, w$] (the value $w$ parametrized, but fixed to 1).
 
 \subsection{Elitism}
 
 A fixed number of top-performing individuals are copied unchanged into the next generation.
 
 \subsection{Mutation}
 
 Several individuals are generated completely at random (random immigrants). This mechanism promotes exploration and maintains population diversity.
 
 \subsection{Offspring Generation}
 
 Three alternative parent-selection strategies are supported:
 
 \begin{itemize}
 	\item \textbf{RKGA}: Two parents are selected uniformly at random, and each gene is inherited from either parent with probability $0.5$.
 	\item \textbf{BRKGA}: One parent is selected from the elite set and one from the non-elite set. Each gene is inherited from the elite parent with a predefined probability.
 	\item \textbf{Lexicase Selection}: Training instances are randomly shuffled. Individuals are filtered iteratively by retaining those achieving the best performance on each instance. Parents are selected randomly from the remaining candidates.
 \end{itemize}
 
 We opt to use the \textit{BRKGA} as the first option to work well. 
 
 
 \subsection{Replacement and Termination}
 
 The new population consists of elites, mutants, and offspring. The algorithm terminates once a predefined time limit is reached (in this case we used 2 hours). The best individual encountered during training is continuously tracked and stored. \textcolor{red}{(Q3) Terminate immediately when the validation value is decreased?}
 
 \section{Overall Learning Procedure}
 
 The learning process alternates between evolutionary optimization of network weights and evaluation through (iterative multi-source) Beam Search. Importantly, the neural network does not directly predict solutions; instead, it learns how to \textit{guide} the search procedure. This places the method within the class of neuro-evolutionary hyper-heuristics.
 
 \section{Algorithm}
 
 \begin{algorithm}[H]
 	\caption{GA-Based Learning of a Beam Search Heuristic}
 	\begin{algorithmic}[1]
 		\STATE Initialize population $P$ with random neural network weights
 		\STATE Evaluate all individuals on training set $T$
 		\STATE Store best solution $\theta^\ast$
 		\STATE $start \leftarrow$ current time
 		\WHILE{time limit $T_{\max}$ has not exceeded}
 		\STATE Sort $P$ by fitness (descending) // {Section 2.1}
 		\STATE Initialize empty population $P'$
 		\STATE Copy top $n_{elites}$ individuals from $P$ to $P'$
 		\FOR{$i = 1$ to $n_{mutants}$}
 		\STATE Create random individual $x$
 		\STATE Evaluate $x$ on $T$
 		\STATE Add $x$ to $P'$
 		\STATE Update $\theta^\ast$ if improved
 		\ENDFOR
 		\FOR{$i = 1$ to $n_{offspring}$}
 		\STATE Select parents using \textbf{RKGA}, BRKGA, or Lexicase
 		\STATE Generate offspring by crossover
 		\STATE Evaluate offspring on $T$
 		\STATE Add offspring to $P'$
 		\STATE Update $\theta^\ast$ if improved
 		\ENDFOR
 		\STATE $P \leftarrow P'$
 		\ENDWHILE
 		\STATE \textbf{return} best weights $\theta^\ast$
 	\end{algorithmic}
 \end{algorithm}
 
 
 \subsection{Explanation of the GA-Based Learning Procedure}
 
 The proposed algorithm employs a Genetic Algorithm (GA) to learn the parameters of a neural network that guides a beam search heuristic. The objective is to optimize a parameter vector $\theta$, representing the neural network weights, such that the induced heuristic achieves high performance on a given training set $T$.
 
 \paragraph{Initialization.}
 The procedure starts by generating an initial population $P$ of candidate solutions, where each individual corresponds to a randomly initialized set of neural network weights. All individuals are evaluated on the training set $T$, yielding a fitness value that reflects the quality of the corresponding heuristic. The best-performing individual found so far is stored as $\theta^\ast$.
 
 \paragraph{Evolutionary Process.}
 The algorithm iteratively improves the population until a predefined time limit $T_{\max}$ is reached. At each iteration, the population is sorted in descending order of fitness, ensuring that higher-quality solutions are prioritized. A new population $P'$ is then constructed using three complementary mechanisms:
 
 \begin{itemize}
 	\item \textbf{Elitism.}  
 	A fixed number $n_{elites}$ of top-performing individuals are directly copied from the current population $P$ to $P'$. This guarantees that the best solutions are preserved across generations and prevents degradation of solution quality.
 	
 	\item \textbf{Mutation (Random Immigrants).}  
 	A set of $n_{mutants}$ new individuals is generated by random initialization. These individuals introduce diversity into the population and help prevent premature convergence. Each mutant is evaluated on the training set $T$ and added to $P'$. If a mutant outperforms the current best solution, $\theta^\ast$ is updated accordingly.
 	
 	\item \textbf{Crossover (Recombination).}  
 	The remaining $n_{offspring}$ individuals are generated via recombination. Parent solutions are selected using a specified selection strategy, such as Random-Key Genetic Algorithm (RKGA), Biased RKGA (BRKGA), or Lexicase selection. Offspring are created by combining genetic material from the selected parents. Each offspring is evaluated and added to $P'$, and $\theta^\ast$ is updated if an improved solution is found.
 \end{itemize}
 
 \paragraph{Population Update and Termination.}
 After generating all individuals, the current population $P$ is replaced by the new population $P'$. This evolutionary process continues until the time limit $T_{\max}$ is reached. The algorithm then returns the best solution $\theta^\ast$, representing the neural network weights that yield the most effective beam search heuristic on the training set.
 
 \paragraph{Key Properties.}
 The proposed approach exhibits several desirable characteristics:
 \begin{itemize}
 	\item \textbf{Exploration--Exploitation Balance:} The combination of elitism, mutation, and crossover ensures a balance between intensification around high-quality solutions and diversification across the search space.
 	\item \textbf{Gradient-Free Optimization:} The GA operates directly on neural network parameters without requiring gradient information, making it suitable for non-differentiable or noisy objective functions.
 	\item \textbf{Anytime Behavior:} The best solution $\theta^\ast$ is continuously updated, allowing the algorithm to return a high-quality solution at any point during execution.
 \end{itemize}

 \section{Discussion}
 
 The proposed framework learns heuristics rather than explicit solutions. The neural network is optimized exclusively through downstream search performance, enabling learning in non-differentiable settings. This tightly coupled integration of evolutionary learning and the classical optimization algorithm Beam Search allows the system to adapt its guidance strategy to the structure of the problem instances.
