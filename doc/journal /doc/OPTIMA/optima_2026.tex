	\section{Introduction} 
	The \textit{Longest Common Subsequence Problem} (LCSP)~\cite{DJUKANOVIC2020106499} is a well-known combinatorial optimization problem with numerous applications in   computational biology, particularly in the analysis of molecular  sequences. Given an arbitrary   set  of input sequences $S=\{s_1,\ldots,s_m\}$ over an alphabet $\Sigma$, the goal is to determine the longest possible subsequence common to all $s_i \in S$. In recent decades, many practically motivated variants of the LCSP have been proposed by considering additional structural or biological constraints such as  the constrained, arc-preserving, repetition-free version~\cite{lafond2023longest,lin2002longest}, among others. In this work we study the \textit{Variable Gapped} LCSP (VGLCSP), introduced in~\cite{penga2011longest} and theoretically examined in~\cite{adamson2023longest}, and experimentally in~\cite{djukanovic2026_eurocast}.  This problem extends the standard LCSP by imposing flexible distance limits (``gaps'') between consecutive symbols in the resulting subsequence, allowing these gaps to vary along the input sequences.
	
	 Formally, for each input sequence $s_i \in S$, a gap function $G_{s_i}\colon \{1,\ldots,|s_i|\}\mapsto \mathbb{N}$ defines the maximum distance allowed between the positions of appearances of consecutive characters for resulting common subsequence in $s_i$. That is, if the characters of a subsequence $s$ appear at the positions $i^1_i < \ldots < i^{{|s|}}_i$ in $s_i$,  the gap constraint is satisfied iff $i^x_i - i^{x-1}_i \le G_{s_i}[i^x_i] + 1$ for each $x=2,\ldots,|s|$. A common subsequence $s$ is considered feasible if the corresponding gap constraints hold across all $s_i \in S$. The VGLCSP represents a flexible and realistic model of sequence alignment, which is particularly useful for comparing DNA or protein sequences with variable structural distances, but also in time series analysis, especially in settings where events are required to occur within specified temporal delays~\cite{lainscsek2015delay}. 
	 
	 Although exact dynamic programming approaches exist for two-sequence case ($m=2$) scenario~\cite{peng2014finding}, their computational complexity becomes prohibitive for larger $m$. To the best of our knowledge, no effective exact and only one approximate approaches for general $m$, a clearly NP-hard problem. The only existing approximate approach is an iterative beam search~\cite{djukanovic2026_eurocast}. This methodology is built on the state-space problem representation and introduced space of root nodes. Each such node is explored in both ways via beam search: expanding current partial solution first  by feasibly appending a sequence as its prefix, and subsequently, in another beam search call,  feasibly extending that solution by concatenating letter-by-letter to its tail. At each iterations, new candidates for root nodes are detected and added in the candidate queue. Given that the methodology provides a scalable alternative to inefficient exact methods, this approach provided first results in the literature for arbitrary number of input sequences $m$. On the other hand, the approach relies on the use of several known hand-crafted (upper bound) heuristics for which there is an evidence from the LCS problem literature that they may offer poor guidance when $m$ is large of $\Sigma$ is small~\cite{DJUKANOVIC2020106499}. In the presence of gap constraints, it could be even worsen.  The second detected research gap concerns lack of practically-motivated instances for the tackled problem as only synthetic instances are generated so far.  Driven by these questions, the first gap is filled by designing a learning iterative beam search, which combines hand-crafted search guidance and the learned guidance into a superior one. To learn a search heuristic, an iterative approach is performed, able to learn the guidance in the iterative beam search approach. Shortly, the heuristic score for each node is represented as an outcome of multi-layered neural network derived in a layer-by-layer forward pass. The weights of neural network model are evolved by a genetic algorithm.  The COP literature is known by utilizing similar (hybrid) methodology when learning heuristics, for example in~\cite{djukanovic2025learning} for the restricted LCS, \cite{reixach2024neural} for the square LCS problem, or~\cite{wu2021learning} for routing problems, to name a few. The difference between these is that their learned heuristics are not further combined with others as we did here into one ensemble method. To address the second research gap, we have derived practically-motivated problem instances by taking (two) biologically-motivated benchmark sets from LCS literature, and generating energy-aware gap constraints at each position of inherited input sequences.

A feed-forward neural network (FFNN) is used to parameterize the heuristic score of each node in the beam search. Instead of relying on gradient-based training, which is shown effective for the (constrained) continuous problems,  the network weights are optimized using a Genetic Algorithm (GA), in particular the popular biased RKGA (BRKGA) approach.  Note that here we deal with a fully non-differentiable search space when training this ML model. This novel ensemble heuristic is clearly a new state-of-the-art guidance, according to the echausive experimental evaluation.

In this case-scenario, gap constraints are designed to reflect on local thermodynamic stability estimated via nearest-neighbor interaction around each nucleotide. In this way, the purpose is to obtain a subsequence with stable molecular structure.

 \section{Iterative Multi-Source Beam Search for GLCS Problem}\label{sec:imsbs}
 
 The methodology of Iterative multi-source beam search (IMSBS) is proposed in~\cite{djukanovic2026_eurocast} and here we only present it in some details. Interested readers are advised to check complete details in the above-cited paper. \\
 
	This approach is derived from the general \textit{state graph} formulation for the VGLCSP. Each node represents a partial feasible subsequence, characterized by the vector of current positions in each input sequence and the subsequence length. Specifically, a state $v=(\mathbf{p}^{L,v}, l^v)$ is induced by a partial solution $s^v$ if: (i) $p^{L,v}_i-1$ is the smallest index such that $s^v$ is a subsequence of the prefix string $s_i[1]\cdots s_i[p^{L,v}_i-1]$; (ii) $l^v=|s^v|$; and (iii) all the gaps $G_{s_i}$ are preserved with $s^v$. A directed arc $\alpha=(v_1 v_2)$, labeled $lett(\alpha)={a} \in \Sigma$, exists between states $v_1$ and $v_2$ if $l^{v_2} = l^{v_1} + 1$ and $s^{v_2} = s^{v_1} \cdot {a}$. To expand a state $v$, all letters that occur in each suffix of $s_i$ defined by respective positions $p^{L,v}_i$  are considered. For each such letter $a$, the minimal positions $p^{L,a}_i \geq p^{L,v}_i$ of its appearances are identified fulfilling $p^{L,a}_i - p^{L,v}_i \le G_{s_i}(p^{L,a}_i)$. The resulting child node---if not suboptimal---is given by $w=(p^{L,a}+1, l^v+1)$. The root node  $r:=((1,\ldots,1),0)$ represents the empty subsequence, while goal/target states correspond to non-extendable feasible subsequences.
	
	 Unlike the classical LCSP, the VGLCSP admits multiple (possibly exponentially many) root nodes as the starting position of a root node is constrained-free.  For example, in sequences $s_1=\texttt{ATGGAAAA}$ and $s_2=\texttt{ATCCAAAA}$ with $G_{s_1}=G_{s_2}=1$, the state $r=((1,1),0)$ cannot reach any state with the position vector $(5,5)$ via transitions, missing an opportunity to reach optimal subsequence $\texttt{AAA}$ is starting the search from $r$. This  motivates the development of  IMSBS approach that works on a space of rooted nodes, which are dynamically generated over iterations and explored based on its priority. BS is a popular meta-heuristic approach~\cite{wiseman2016sequence} that performs in a breadth-first-search manner by expanding up to a certain number of nodes at the current level until the bottom is reached. 
	 This amount of nodes is controlled by the size of beam ($\beta$)---which controls the level of greediness---and a heuristic guidance ($h$) for selecting them---as especially sensible algorithmic parameter. 
	 
	 
	 In the IMSBS, BS is iteratively executed with a set of promising candidate root nodes as an initial beam, selected from a dynamically maintained global priority queue of candidate root nodes, controlling the diversification of the approach.	Each iteration performs a complete BS on a few  ($num\_root\_nodes$) top candidate root nodes, which are, beforehand, refined by the beam search with a small beam width $\beta'$ and heuristic guidance $h'$, as not all candidate nodes indeed represent root nodes. This process intensify the search and permit it from returning proven suboptimal solutions. During each BS call, new candidate root nodes are generated from the complete (non-expandable) nodes and subsequently added in the priority queue. This approach balances the intensification and diversification by several of its parameters. The stopping criteria for the IMSBS are the number of allowed beam search iterations ($bs\_iters$), and time limit of 30 minutes.
	 
	 In the remaining part of the paper, we refer to the algorithm as \texttt{IMSBS}($\beta, h, \beta'$, $h', num\_root\_nodes$), returning an approximate solution $s_{imsbs}$. 
 
 
 \section{Learning to Guide the Search} \label{sec:ffnn_guidance}
 
 Neural networks are well-known machine learning model~\cite{gurney2018introduction} that has shown its applicability in majority of fields~\cite{bishop1994neural}. There are many kind of neural networks such as the basic one, multi-layer perceptron, towards more complex Constitutional and recurrent neural networks of different architectures~\cite{mienye2024recurrent}.  We will here keep the things simple, and will consider only the fully-connected (multi)layered neural networks that will serve as tool to guide the search process of our iterative beam-search approach instead of the hand-crafted versions used in~\cite{djukanovic2026_eurocast}. For doing so, the major issue is to train the neural network, that is, setting its proper weights. In the context of classical neural networks for regression, this process is performed by alternating between forward and backward propagation, where the later is used to adapt the weights to minimize the error score on a set of  training data w.r.t.\ a loss function. The process of weights adaptation is usually executed by employing iterative numerical procedures such as (stohastic) gradient descent~\cite{amari1993backpropagation} or versions thereof using the power of gradient in quickly reaching local optima and decreasing the overall loss w.r.t. known target values, hopping to generalize well on unseen data. \\
 
 
 As we operate in combinatorial space of an \textsc{NP}-hard problem, we do not expect in general target values are known in advance, or to some extent these will be known but only for small-to-middle sized instances if an efficient exact solver exists. This limits the application of gradient descent algorithms in learning weights for such environments. Thus, we approach here differently, by employing meta-heuristic approach with a diverse population of solutions representing weights of NN, able to quickly evolve towards local optima. Therefore, the back propagation pass in the classical NN training process is replaced by a population based meta-heuristic which adopts weights over iterations. In the feed forward pass fitness value for specific NN configuration has been determined. \\
 
  
\textbf{Predefined Architecture of NN}. The heuristic score is represented by a \textit{multi-layer perceptron} with an number of hidden layers and user-defined units per layer, the standard input and output layer.  The input layer received a set of numerical values associated to subproblem to be evaluated, and the output consists of one (real) value, indicating how the considered subproblem is ``important''. 
At each layer, to catch up non-linear relations between input features,  we support the use of following activation functions: \texttt{hyperbolic tangent}, \texttt{ReLU}, and \texttt{sigmoid}.
 
 For each layer $l$, forward propagation is defined as:
 \[
 \mathbf{z}^{(l+1)} = W^{(l)} \mathbf{a}^{(l)} + b^{(l)}, \quad
 \mathbf{a}^{(l+1)} = \phi(\mathbf{z}^{(l+1)}),
 \]
 where $W^{(l)}$ and $b^{(l)}$ are the weight matrix and bias vector, respectively, and $\phi$ denotes the activation function.  All weights and biases are flattened into a \textbf{single vector}, which forms the chromosome of a GA individual. \\
 
 \textbf{Fitness Evaluation of NN configurations}. Given a candidate weight vector $w$, we perform:  1) Load of the weights into a FFNN $\mathcal{N}(w)$ of fixed architecture;
 	2) Executing the IMSBS on each instance from a training set $\mathcal{T}$ utilizing the neural heuristic (to guide the search in forward iterations of IMSBS as a sensible step); 3)  Recording size of the best sequence obtained for each instance.
 
 The fitness value is computed as the average solution quality over all training instances. Formally, the learning problem is defined as:
 \begin{equation} \label{eq:training_problem_optimization}
     \max_{w} \; \emph{Fitness}(w)=\frac{1}{|T|} \sum_{I \in \mathcal{T}} \texttt{IMSBS}(I, h_w),
 \end{equation}
 where $h_w$ is the heuristic induced by neural network $\mathcal{N}(w)$. Validation performance score is computed analogously on a separate dataset $\mathcal{V}$ and is used only for reporting.  
 
 
\textbf{Feature Extraction for the VGLCSP}. To guide the search process, each node is represented by a fixed-length numerical feature vector capturing both the current state in construction phase and the structure of induced subproblem to solve. These features are designed to be instance-independent. %and comparable across problem sizes.
 
 Let $S=\{s_1,\ldots, s_k\}$ denote the set of input sequences, and let $v=(\textbf{p}^v, l^v)$ be  a given node. The vector $\mathbf{p}^L=(\textbf{p}^v_1,\ldots,\textbf{p}^v_m)$ is first normalized by the corresponding sequence lengths $F^{v} = (F^{v})_i \leftarrow \frac{\textbf{p}^v_i}{|s_i|}, i=1, \ldots, m$ 
 so that progress in each sequence lies in $[0,1]$ and becomes independent of absolute input sizes.
 
 From this normalized position vector, four statistical descriptors are extracted: maximum, minimum, mean, and standard deviation (4 features). These quantities characterize, respectively, the most advanced and least advanced sequences, the average depth of the search, and the imbalance of progress among sequences. In addition, \textit{the current length of partially constructed subsequence} $l^v$ is included as a feature (the fifth feature). Together, these values provide a compact description of the current search state. \\
 
 To capture the feasibility of future extensions, gap-related features are computed from the positions of remaining sequences. That is, for each $v$,  we determine $G^{\min} = \min\{ G_i(j) \mid j=\textbf{p}_i^v, i=1, \ldots, m \}$,  $G^{\max} =\max\{ G_i(j) \mid j=\textbf{p}_i^v, i=1, \ldots, m \}$,  $G^{avg}_i = \text{mean}\{ G_i(j) \mid j=\textbf{p}_i^v, i=1, \ldots, m \}$, and  
 $G^{\text{std}}_i = \text{std}\{ G_i(j) \mid j=\textbf{p}_i^v, i=1, \ldots, m \}$, where $G_1, \ldots, G_m$ are corresponding gap constraints associated to $s_1, \ldots, s_m$ respectively.
 %\textcolor{red}{we speed up the calculation of G-stats -- preprocessing steps (not explained here due to page limit constraints)... }. 
This gives us additionally 4 features; in overall 9 features are comprised to create the \textbf{feature configuration 1} (F1).  These statistics for gap constrains summarize the remaining flexibility of the instance: large average or maximum gaps indicate greater freedom for future matches, whereas small minimum gaps and large variability typically signal tighter constraints and increased difficulty.
 Additional feature $|\Sigma|$ creates the \textbf{feature configuration 2} (F2); When adding the value of $m$ (number of input sequences) to F2, we obtain the \textbf{feature configuration 3} (F3).   
 %Depending on the selected configuration, additional instance-level characteristics are appended to the feature vector. These may include 
  The alphabet size $|\Sigma|$ and the number of sequences $(m)$ as the attributes provide global structural information about the instance and help the learning model adapt to varying problem scales. Note that we additionally probed with more features such as length of input sequences, but this has negative impact on solution quality of the learning approach. 
 
 Finally, all features are standardized to zero mean and unit variance prior to being passed to the learning component. The utilization of normalization is know to contribute decreased sensibility of learning process in the process of training ML models.  
 Overall, the resulting representation combines information about (i) the current construction progress, (ii) the remaining gap-constraint landscape, and (iii) global instance properties, yielding between 9 and 11 features depending on the (3) configuration. This compact yet expressive encoding enables effective learning-based guidance of the search for high-quality  subsequences. \\
 
 \textbf{Optimizer of NN Configurations}. We opt for Genetic Algorithm, in particular its popular version called the Biased Random Key Genetic Algorithm (BRKGA) to solve the problem given in Eq.~(\ref{eq:training_problem_optimization}).  The optimization process follows a population-based evolutionary scheme, described in Algorithm~\ref{alg:learning_mechanism}. Before we briefly explain the main workflow of the algorithm, first a few important notes. 
 
  Each individual is initialized by sampling weights uniformly from a predefined interval [$-w, w$] ($w$ is parametrized, but here fixed to 1).
  \textit{Elitism} is included by copying a fixed number of top-performing individuals into the next generation. \textit{Mutation} is performed so that several individuals are generated completely at random (random immigrants). This mechanism promotes exploration and maintains population diversity.
 
 Concerning \textit{offspring Generation} in BRKGA, 
 one parent is selected from the elite set and one from the non-elite set. Each gene is inherited from the elite parent with a predefined probability (we fix it to 0.5).
 
 %We opt to use the \textit{BRKGA} as the first option to work well. 
 
 The new population consists of elites, mutants, and offspring. The algorithm terminates once a predefined time limit is reached (in this case we used 2 hours). The best individual encountered during training is continuously tracked and stored. Once the new incumbent is reached, its validation score is also determined. If it is decreased in comparison to the best so far validation value, over-fitting is detected and the learning process is immediately terminated. 
 
 
 \textbf{Overall Learning Procedure}. The learning process alternates between evolutionary optimization of network weights and evaluation through (iterative multi-source) Beam Search. Importantly, the neural network does not directly predict solutions; instead, it learns how to \textit{guide} the search procedure. This places the method within the class of neuro-evolutionary hyper-heuristics.
 
 \begin{algorithm}[H]
 	\caption{GA-Based Learning of a BS Heuristic} \label{alg:learning_mechanism}
 	\begin{algorithmic}[1]
 		\STATE \textbf{Input}: NN architecture, parameters of IMSBS for \emph{Fitness}, parameters of BRKGA
 		\STATE \textbf{Output}: best weights for NN 
 		\STATE Initialize population $P$ with random neural network weights
 		\STATE Evaluate all individuals on training set $\mathcal{T}$
 		\STATE Store best solution $w^\ast$
 		\STATE $start \leftarrow$ current time
 		\WHILE{time limit $t_{\max}$ has not exceeded}
 		\STATE Sort $P$ by \emph{Fitness}($\cdot$) value (descending)
 		\STATE Initialize empty population $P'$
 		\STATE Copy top $n_{elites}$ individuals from $P$ to $P'$
 		\FOR{$i = 1$ to $n_{mutants}$}
 		\STATE Create random individual $x$
 		\STATE Evaluate $x$ on $\mathcal{T}$
 		\STATE Add $x$ to $P'$
 		\STATE Update $w^\ast$ if improved \& validation score not decreased 
 		\ENDFOR
 		\FOR{$i = 1$ to $n_{offspring}$}
 		\STATE Select parents from the elite set and the set of rest of individuals 
 		\STATE Generate offspring by crossover
 		\STATE Evaluate offspring on $\mathcal{T}$
 		\STATE Add offspring to $P'$
 		\STATE Update $w^\ast$ if improved  
 		\ENDFOR
 		\STATE $P \leftarrow P'$
 		\ENDWHILE
 		\STATE \textbf{return} $w^\ast$
 	\end{algorithmic}
 \end{algorithm}

The objective is to optimize a parameter vector $\theta$, representing the neural network weights, such that the induced heuristic achieves high performance on a given training set $T = \{I_1, I_2, \dots, I_m\}$ of problem instances.

\paragraph{Fitness Function.}
Let $\mathcal{B}(I, \theta)$ denote the solution obtained by applying a beam search guided by parameters $\theta$ on instance $I$. Let $f(\cdot)$ be a problem-specific objective function (e.g., solution quality or cost). The fitness of an individual $\theta$ is defined as:
\begin{equation}
	F(\theta) = \frac{1}{|T|} \sum_{I \in T} f\big(\mathcal{B}(I, \theta)\big).
\end{equation}
Depending on the problem formulation, $f$ can represent either a maximization objective (e.g., solution value) or a minimization objective (e.g., cost), in which case an appropriate transformation is applied to ensure consistency.

\paragraph{Beam Search Integration.}
The beam search procedure maintains, at each depth level $t$, a set of at most $k$ partial solutions (beam width $k$). Let $\mathcal{S}_t$ denote the set of candidate states at level $t$. For each state $s \in \mathcal{S}_t$, a feature representation $\phi(s)$ is computed and evaluated by the neural network:
\begin{equation}
	h_\theta(s) = \text{NN}_\theta(\phi(s)),
\end{equation}
where $h_\theta(s)$ represents the heuristic score assigned to state $s$. The beam is then updated by selecting the top-$k$ states according to $h_\theta(s)$:
\begin{equation}
	\mathcal{S}_{t+1} = \text{Top}_k \left\{ s' \mid s' \in \text{Expand}(\mathcal{S}_t),\; h_\theta(s') \right\}.
\end{equation}
This process continues until a terminal state is reached, yielding the final solution $\mathcal{B}(I, \theta)$.

\paragraph{Evolutionary Process.}
The algorithm iteratively improves the population until a predefined time limit $T_{\max}$ is reached. At each iteration, the population is sorted in descending order of fitness. A new population $P'$ is constructed using three mechanisms:

\begin{itemize}
	\item \textbf{Elitism.}  
	A fixed number $n_{elites}$ of top-performing individuals are directly copied from $P$ to $P'$, preserving the best solutions.
	
	\item \textbf{Mutation (Random Immigrants).}  
	A set of $n_{mutants}$ individuals is generated randomly to introduce diversity. Each is evaluated using $F(\theta)$ and added to $P'$, updating $\theta^\ast$ if improved.
	
	\item \textbf{Crossover (Recombination).}  
	The remaining $n_{offspring}$ individuals are generated via recombination. Parents are selected using strategies such as RKGA, BRKGA, or Lexicase selection. Offspring inherit characteristics from both parents, are evaluated, and added to $P'$, with updates to $\theta^\ast$ if necessary.
\end{itemize}

\paragraph{Population Update and Termination.}
After generating all individuals, the population is updated as $P \leftarrow P'$. The process repeats until the time limit $T_{\max}$ is reached. The algorithm returns the best solution $\theta^\ast$, corresponding to the neural network weights that induce the most effective beam search heuristic on the training set.

\paragraph{Key Properties.}
\begin{itemize}
	\item \textbf{Exploration--Exploitation Balance:} The combination of elitism, mutation, and crossover ensures both intensification and diversification.
	\item \textbf{Gradient-Free Optimization:} The approach does not rely on gradient information and is suitable for non-differentiable objectives.
	\item \textbf{Anytime Behavior:} The best solution $\theta^\ast$ is continuously updated and available at any point during execution.
\end{itemize}

 \section{Discussion}
 
 The proposed framework learns heuristics rather than explicit solutions. The neural network is optimized exclusively through downstream search performance, enabling learning in non-differentiable settings. This tightly coupled integration of evolutionary learning and the classical optimization algorithm Beam Search allows the system to adapt its guidance strategy to the structure of the problem instances.
