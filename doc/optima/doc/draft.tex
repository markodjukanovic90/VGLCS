\section{Introduction}

This work proposes a learning framework for automatic constructing a data-driven heuristic to guide Beam Search in combinatorial optimization problems, specifically demonstrated for a sequence problem that arises in bioinformatics, the \textit{Variable Gapped Longest Common Subsequence Problem} (VGLCSP). A feed-forward neural network (FFNN) is used to parameterize the heuristic function, in particular as its outcome provide through the feed-forward pass.  Instead of relying on gradient-based training which is mainly used in the context of (constrained) continuous problems minimizing the error in an iterative procedure by using numerical methods such as gradient descent, the network weights are optimized here using effective (population-based) meta-heuristics such as Genetic Algorithm (GA), precisely its popular version  called the  biased RKGA (BRKGA) approach.  

Each individual in the GA population represents a complete set of neural network weights. The quality of an individual is evaluated by embedding the neural network inside the \textit{Iterative Multi-source Beam Search} (IMSBS) framework (proposed in the paper for EUROCAST 2026) to guide the search in the process of  forward passes of the BS,  
and measuring the average solution quality obtained on a set of training instances. Validation instances are used exclusively to monitor generalization.

Formally, the learning problem is defined as:
\[
\max_{\theta} \; \frac{1}{|T|} \sum_{I \in T} f(\text{IMS\_BeamSearch}(I, h_\theta)),
\]
where $T$ denotes the training set, $h_\theta$ is the neural heuristic parameterized by weights $\theta$, and $f(\cdot)$ measures solution quality.

\section{Neural Network Model}

The heuristic is represented by a \textit{multilayer perceptron} with an arbitrary number of hidden layers and user-defined units per layer. Supported activation functions include \texttt{hyperbolic tangent}, \texttt{ReLU}, and \texttt{sigmoid}.

For each layer $l$, forward propagation is defined as:
\[
\mathbf{z}^{(l+1)} = W^{(l)} \mathbf{a}^{(l)} + b^{(l)}, \quad
\mathbf{a}^{(l+1)} = \phi(\mathbf{z}^{(l+1)}),
\]
where $W^{(l)}$ and $b^{(l)}$ are the weight matrix and bias vector, respectively, and $\phi$ denotes the activation function.

All weights and biases are flattened into a single vector, which forms the chromosome of a GA individual.

\subsection{Fitness Evaluation}

Given a candidate weight vector:
\begin{enumerate}
    \item The weights are loaded into the FFNN.
    \item IMSBS is executed on each training instance using the neural heuristic (guiding the search in the forward iterations as more sensible than the backward iterations).
    \item The size of the best sequence obtained for each instance is recorded.
\end{enumerate}

The fitness value is computed as the average solution quality over all training instances. Validation performance is computed analogously on a separate dataset and is used only for reporting.

\subsection{Feature Extraction for the Variable-Gap LCS}

To guide the search process, each partial solution (node) is represented by a fixed-length numerical feature vector capturing both the current state of the construction and the structure of the remaining search space. These features are designed to be instance-independent and comparable across problem sizes.

Let $S=\{s_1,\ldots, s_k\}$ denote the set of input sequences, and let $p_i$ be the current matched position in sequence $s_i$ at a given node. The vector $\mathbf{p}^L=(p_1,\ldots,p_k)$ is first normalized by the corresponding sequence lengths,
\[
p_i \leftarrow \frac{p_i}{|s_i|},
\]
so that progress in each sequence lies in $[0,1]$ and becomes independent of absolute input sizes.

From this normalized position vector, four statistical descriptors are extracted: maximum, minimum, mean, and standard deviation (4 features). These quantities characterize, respectively, the most advanced and least advanced sequences, the average depth of the search, and the imbalance of progress among sequences. In addition, \textit{the current length of the partially constructed subsequence} is included as a feature (as the fifth feature). Together, these values provide a compact description of the current search state.

To capture the feasibility of future extensions, gap-related features are computed from the positions of remaining sequences (subproblem) of input sequences. For each sequence $s_i$, all gap bounds associated with positions strictly after the current node position are collected. Symbolically, for each $v$,  $Gap^{min}_i$ is calculated as $\min\{ G_i(j) \mid j=p_i^L, i=1, \ldots, m \}$,  $Gap^{max}_i$ is calculated as $\max\{ G_i(j) \mid j=p_i^L, i=1, \ldots, m \}$,  $Gap^{avg}_i$  as $\text{AVG}\{ G_i(j) \mid j=p_i^L, i=1, \ldots, m \}$, and  
 $Gap^{std}_i$  as $\text{STD}\{ G_i(j) \mid j=p_i^L, i=1, \ldots, m \}$, where $G_1, \ldots, G_m$ are gap constraints associated to $s_1, \ldots, s_m$ respectively. This gives us additionally 4 features; in overall 9 features are used for the \textbf{feature configuration 1} (F1).  These statistics for gap constrains summarize the remaining flexibility of the instance: large average or maximum gaps indicate greater freedom for future matches, whereas small minimum gaps and large variability typically signal tighter constraints and increased difficulty.
  Additional feature $|\Sigma|$, this 10 features, created the \textbf{feature configuration 2} (F2); When adding $m$ (number of input sequences) to F2, we obtain \textbf{feature configuration 3} (F3). 
 
%Depending on the selected configuration, additional instance-level characteristics are appended to the feature vector. These may include 

The alphabet size $|\Sigma|$ and the number of sequences $(m)$ as the attributes provide global structural information about the instance and help the learning model adapt to varying problem scales.

Finally, all features are standardized to zero mean and unit variance prior to being passed to the learning component. The utilization of normalization is know to contribute decreased sensibility of learning process when ML models are trained.  
Overall, the resulting representation combines information about (i) the current construction progress, (ii) the remaining gap-constraint landscape, and (iii) global instance properties, yielding between 9 and 11 features depending on the configuration. This compact yet expressive encoding enables effective learning-based guidance of the search for high-quality variable-gap longest common subsequences.
 
\section{Genetic Algorithm}

The optimization process follows a population-based evolutionary scheme.

\subsection{Initialization}

Each individual is initialized by sampling weights uniformly from a predefined interval [$-w, w$] (the value $w$ parametrized, but fixed to 1).

\subsection{Elitism}

A fixed number of top-performing individuals are copied unchanged into the next generation.

\subsection{Mutation}

Several individuals are generated completely at random (random immigrants). This mechanism promotes exploration and maintains population diversity.

\subsection{Offspring Generation}

The best parent-selection strategy chosen:

\begin{itemize}
   % \item \textbf{RKGA}: Two parents are selected uniformly at random, and each gene is inherited from either parent with probability $0.5$.
    \item \textbf{BRKGA}: One parent is selected from the elite set and one from the non-elite set. Each gene is inherited from the elite parent with a predefined probability.
   % \item \textbf{Lexicase Selection}: Training instances are randomly shuffled. Individuals are filtered iteratively by retaining those achieving the best performance on each instance. Parents are selected randomly from the remaining candidates.
\end{itemize}

We opt to use the \textit{BRKGA} as the first option to work well. 


\subsection{Replacement and Termination}

The new population consists of elites, mutants, and offspring. The algorithm terminates once a predefined time limit is reached (in this case we used 2 hours). The best individual encountered during training is continuously tracked and stored. We employed an early termination by monitoring the validation value. In details, if a new  incumbent configuration has reached, we determine the validation quality. If the value is decreased acc. to the best so far validation quality, the process of learning terminates by reporting overfitting. 

\section{Overall Learning Procedure}

The learning process alternates between evolutionary optimization of network weights and evaluation through (iterative multi-source) Beam Search. Importantly, the neural network does not directly predict solutions; instead, it learns how to \textit{guide} the search procedure. This places the method within the class of neuro-evolutionary hyper-heuristics.

\section{Algorithm}

\begin{algorithm}[H]
\caption{GA-Based Learning of a Beam Search Heuristic (BRKGA)}
\begin{algorithmic}[1]
\STATE Initialize population $P$ with random neural network weights
\STATE Evaluate all individuals on training set $T$
\STATE Store best solution $\theta^\ast$
\STATE $start \leftarrow$ current time
\WHILE{time limit $T_{\max}$ has not exceeded}
    \STATE Sort $P$ by fitness (descending) // {Section 2.1}
    \STATE Initialize empty population $P'$
    \STATE Copy top $n_{elites}$ individuals from $P$ to $P'$
    \FOR{$i = 1$ to $n_{mutants}$}
        \STATE Create random individual $x$
        \STATE Evaluate $x$ on $T$
        \STATE Add $x$ to $P'$
        \STATE Update $\theta^\ast$ if improved
    \ENDFOR
    \FOR{$i = 1$ to $n_{offspring}$}
        \STATE Select parents using, BRKGA
        \STATE Generate offspring by crossover
        \STATE Evaluate offspring on $T$
        \STATE Add offspring to $P'$
        \STATE Update $\theta^\ast$ if improved
    \ENDFOR
    \STATE $P \leftarrow P'$
\ENDWHILE
\STATE \textbf{return} best weights $\theta^\ast$
\end{algorithmic}
\end{algorithm}

As the best optimizer, the BRKGA setting used in training: $n_{pop}=20, n_{elites}=1, p_{e}=0.5, p_m=7$.

\subsection{Discussion}


The proposed framework learns heuristics rather than explicit solutions. The neural network is optimized exclusively through downstream search performance, enabling learning in non-differentiable settings. This tightly coupled integration of evolutionary learning and the classical optimization algorithm Beam Search allows the system to adapt its guidance strategy to the structure of the problem instances.


\subsection{Improvements: an ensemble method}

To guide the beam search, we employ a rank-based ensemble strategy that combines a learned heuristic with an analytical upper bound (UB) estimate. Specifically, each candidate node is evaluated using two criteria: (i) a neural score obtained from a trained feed-forward neural network (FFNN), and (ii) a problem-specific upper bound ($\text{UB}_2$ in the EUROCAST paper). 

Instead of combining the raw scores directly, which may differ in scale and distribution, we adopt a rank aggregation approach. First, candidate nodes are sorted independently according to each criterion, producing two rankings. Each node is then assigned a rank based on its position in each sorted list. 

The final ranking score is computed as a weighted combination of the two ranks:
\[
r(v) = \alpha \cdot r_{\text{FFNN}}(v) + (1 - \alpha) \cdot r_{\text{UB}}(v),
\]
where $\alpha = 0.45$ in our implementation except for $m=3$ where $\alpha=0.5$ is fixed. Nodes are then sorted in ascending order of $r(v)$, and the top candidates are retained according to the beam width.

This rank-based ensemble provides a robust mechanism for integrating learned and analytical guidance, mitigating scale inconsistencies and improving search stability.

 \subsection{Structure-Aware Benchmark Generator for Multi-Sequence Gapped LCS}
 
 We propose a benchmark generator for the multi-sequence gapped longest common subsequence (GLCS) problem, where position-dependent gap constraints are derived from local structural influence.
 
 Given a set of sequences:
 $
 \mathcal{S} = \{S_1, S_2, \dots, S_m\},
 $
 each sequence $S_j = (s_j[1], \dots, s_j[n])$ is associated with gap constraints:
 $
 w^{(j)} = (w^{(j)}_1, \dots, w^{(j)}_n),
 $
 where $w^{(j)}_i$ determines the maximum number of symbols that may be skipped before matching position $i$.

\subsection{Dinucleotide-Based Structural Model}

We define a dinucleotide weight function:
$$
f(s_i, s_{i+1}) = -\Delta G(s_i, s_{i+1}),
$$
where $\Delta G$ denotes the nearest-neighbor free energy.

\paragraph{Causal positional contribution.}

To ensure consistency with the directional nature of the gapped LCS constraint, we define a \emph{causal} positional contribution. Each position $k$ is associated only with the interaction formed with its immediate predecessor:

$
f^{(j)}_k =
\begin{cases}
	f(s^{(j)}_{k-1}, s^{(j)}_k), & k > 1, \ \\
	0, & k = 1.
\end{cases}
$

\textit{Interpretation.} \begin{itemize}
	\item Each position inherits structural influence from the bond linking it to the previous nucleotide (linkage between these considered).
	\item First position has no predecessor and is therefore assigned zero contribution.
\end{itemize}

\paragraph{Rationale.}
This definition enforces a left-to-right dependency structure, ensuring that all subsequent computations of influence and gap constraints depend only on previously observed positions. This is consistent with the feasibility condition in the gapped LCS problem, where admissibility of extending a subsequence depends on the last selected position.

\paragraph{Remark.}
Compared to centered models, the causal formulation removes any dependence on future positions, resulting in a strictly sequential and history-aware structural representation.

 
\subsection{Causal (Left-Sided) Spatial Influence Aggregation}

In order to align the influence model with the semantics of the gapped LCS constraint, we define influence in a \emph{causal} (left-to-right) manner.

The influence at position $i$ is computed using only preceding positions:
$$
I^{(j)}(i) = \sum_{k=\max(1,i-r)}^{i} e^{-\lambda (i-k)} \cdot f^{(j)}_k.
$$

\paragraph{Interpretation.}
\begin{itemize}
	\item Only positions $k \leq i$ contribute to the influence of $i$.
	\item The exponential kernel assigns higher weight to positions closer to $i$.
	\item The parameter $r$ limits the historical window.
\end{itemize}

\paragraph{Rationale.}
This formulation reflects the fact that, in the gapped LCS problem, feasibility of selecting position $i$ depends on the previously selected position. Therefore, the structural context relevant to $i$ must be derived exclusively from preceding positions.

\paragraph{Consequence.}
The resulting influence values induce \emph{directional structure} in the sequence:
\begin{itemize}
	\item regions preceded by strong interactions accumulate high influence,
	\item regions following weak or irregular segments exhibit lower influence.
\end{itemize}

  
 \textit{Normalization}. 
 
 $$
 \hat{I}^{(j)}_i = \frac{I^{(j)}_i - \min(I^{(j)})}{\max(I^{(j)}) - \min(I^{(j)}) + \epsilon} \in \left[0, 1 \right].
$$
 
 \subsection{Direct Mapping to Gap Constraints}
 
 Gap constraints are obtained via a \emph{direct mapping}:
 $$
 w^{(j)}_i = w_{\min} + (w_{\max} - w_{\min}) \cdot \hat{I}^{(j)}_i.
 $$
 
 \paragraph{Interpretation.}
 \begin{itemize}
 	\item high influence $\Rightarrow$ large allowed gap,
 	\item low influence $\Rightarrow$ restrictive gap.
 \end{itemize}
 
 This mapping reflects the idea that positions with strong structural connectivity can support longer-range dependencies and allow more flexible matching.

 \textit{Discussion}. The proposed generator creates structured instances where:
 \begin{itemize}
 	\item high-influence regions act as \emph{flexible hubs},

 \subsection{Results on RAT \& VIRUS benchmark set}
 
 To demonstrate effectiveness of the two approaches on real-world instances, we have generated a set of instances by using well-known benchmark sets \textsc{Rat} and \textsc{Virus} from the LCS problem literature. In details, we consider only those instances with $|\Sigma|=4$, as for the instances with $|\Sigma|=20$, the results were rather short solutions---therefore not relevant from the biological standpoint---  generating gaps in the same way as for the synthetic instances.  On each input sequences in each instances, we have generated one gap constraint by the methodology introduced in this chapter. In this way, input sequences are copied in corresponding VGLCS instances, while gap constraints have been accompanied  after generation. This dataset consists of $2\cdot 10=20$ problem instances. All sequences are of length 600, and, as already mentioned, over an alphabet set of size 4. Their differ characteristically by $m$ values which range from 10 to 200. In our experiments, we have taken the settings of \textsc{Imsbs} used in for synthetic instances, and for the \textsc{Limsbs}-ensemble, a neural network trained specifically for a largest our $m$ because in the case of these 20 instances,  $m\geq 10$ holds.

 The following conclusions can be made form the obtained results, reprted in
 Table~\ref{tab:real_results}. The results indicate a clear advantage of LIMSBS in terms of solution quality. 
 \begin{itemize}
 	\item  Overall, LIMSBS outperforms IMSBS in 15 out of 20 instances, while 5 instances result in ties and none favor IMSBS. This demonstrates a consistent dominance of LIMSBS across the tested dataset.
 
    \item In terms of solution quality, LIMSBS achieves an average value of 11.8, compared to 9.9 for IMSBS, corresponding to an average relative improvement of approximately 19\%. The median quality values are 5 (LIMSBS) and 4 (IMSBS), confirming that improvements are consistent and not driven by outliers.
 
   \item The advantage of LIMSBS is particularly pronounced on more difficult instances (e.g., virus cases with higher variability), where substantial gains are observed (e.g., 87 vs.\ 36). In contrast, ties typically occur on simpler instances with very low objective values.
 
   \item Regarding computational time, IMSBS is generally faster, achieving an average runtime of 36.9 seconds compared to 32.6 seconds for LIMSBS. However, this difference is not consistent across all instances, and in several cases LIMSBS is competitive or even faster.
  \end{itemize}
  The results again demonstrate that LIMSBS provides a substantial and consistent improvement in solution quality over IMSBS on real-world instances, at the cost of a comparable computational effort. This makes LIMSBS a preferable choice when solution quality is the primary objective.
  
  \section{Future Work}  
  
  \textbf{The training process}. Using only the mean optimizes average performance, ignores failures, and can produce unstable heuristics.  In COPs, robustness is often more important than average performance.
  
  Fitness function replacement:
  \begin{itemize}
  	\item  Option A (robust \& simple)   Fitness = 0.7 $\cdot$ mean + 0.3 $\cdot$ 20th-percentile
  	\item Option B (robust learning):    Fitness = 20-th percentile 
  	\item allow some degradation in the validation fitness score, not immediately stop by obtaining a weaker score.  
  \end{itemize}
  
  \textbf{ Curriculum / weighted fitness}.   Give more importance to harder instances and recent failures. 
  Fitness=$\sum_i w_i f_i$, e.g., $w_i = \frac{1}{2^i}$, or $w_i=\frac{1}{n+1}$.
